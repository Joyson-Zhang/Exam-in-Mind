## 2026-04-12

### `fix(renderer): MkDocs 站点 file:// 协议下链接无法跳转` · commit `9e4b5dc`
**问题**: 双击 index.html 打开站点时，侧边栏链接跳转到文件系统目录索引而非页面内容。
**做法**: 在 mkdocs_renderer.py 中设置 `use_directory_urls: false`，让链接直接指向 .html 文件。
**效果**: ✅ 侧边栏链接可在 file:// 协议下正常跳转。

### `fix(content): formulas 数组字符串展平 + LaTeX 反斜杠保护` · commit `8aabff5`
**问题**: Claude 返回的 formulas 中 62 个叶子节点的公式被整体序列化为一个字符串；首次修复使用 ast.literal_eval 导致 LaTeX 中 `\t`→tab、`\f`→formfeed 等 267 个公式被控制字符损坏。
**做法**: 改用"先转义所有反斜杠再 json.loads"策略替代 ast.literal_eval；修复已损坏的 tree.json 数据（控制字符→对应 LaTeX 反斜杠序列）。
**效果**: ✅ 734 个公式全部正常，0 个控制字符残留。

### `fix(renderer): footer 遮挡侧边栏底部内容` · commit `f9b7408`
**问题**: MkDocs Material footer（z-index 高于侧边栏）在 file:// 协议下遮挡侧边栏底部导航条目；首次修复用 `z-index: 1` 不足以解决 stacking context 冲突。
**做法**: 直接 `display: none` 隐藏 footer（"Made with Material for MkDocs" 对本地知识树无实际用途）。
**效果**: 🔄 待用户验证。

### `fix(renderer): 公式定界符归一化` · commit `f9b7408`
**问题**: tree.json 中部分公式缺少闭合 `$`（如 `$formula` 无结尾 `$`），渲染器仅检查开头有无 `$` 来决定是否包裹，导致缺闭合的公式被原样输出为纯文本；首次修复采用"剥离再重包裹"策略，误伤了 `$formula$ (说明文字)` 格式产生多余 `$`。
**做法**: 重写 `_normalize_formula()`: 检测是否已有配对定界符→有则原样返回；缺闭合→补上；完全无→包裹 `$...$`。
**效果**: 🔄 待用户验证。

### `fix(renderer): definition 中字面量 \n 未转换为换行` · commit `2d7b4bd`
**问题**: 14 个叶子节点的 definition 中 LLM 返回了字面量 `\n`（两个字符），未被转换为真正换行符，在页面中显示为可见的 `\n\n1.` 等文本。
**做法**: 新增 `_unescape_literal_newlines()`，用负向前瞻正则 `\\n(?![a-zA-Z])` 替换字面量 `\n` 为换行符，同时避免误伤 LaTeX 命令（`\neq`、`\nu` 等）。
**效果**: 🔄 待用户验证。

### `fix(renderer): 侧边栏底部导航条目不可见` · commit `2d7b4bd`
**问题**: 隐藏 footer 时一并移除了侧边栏底部内边距，导致最后几个导航条目无法滚动到可见区域。
**做法**: 在 custom.css 中为 `.md-sidebar__inner` 添加 `padding-bottom: 3rem`。
**效果**: 🔄 待用户验证。

### `feat(renderer): 顶部 tab 导航栏滚动箭头` · commit `2207a99`
**问题**: MkDocs Material 的 `navigation.tabs` 在章节较多时溢出，但原生滚动条被隐藏（`scrollbar-width: none`），用户难以发现可横向滚动。
**做法**: 新增 `tab-scroll.js` + CSS，在 tab 栏两端动态注入 ◀▶ 箭头按钮；点击平滑滚动 200px；根据滚动位置自动显示/隐藏；支持亮/暗色模式。
**效果**: ✅ 用户验证通过。

### `fix(renderer): 侧边栏无法滚动显示全部导航链接` · commit `a2932e3`
**问题**: Material 主题桌面端设置 `.md-sidebar { height: 0 }`（依赖 JS 动态计算）、移动端设置 `.md-sidebar__scrollwrap { overflow: hidden }`，在 file:// 协议下两者均导致侧边栏无法滚轮滚动。
**做法**: 在 custom.css 中用 `!important` 覆盖：桌面端 `height: calc(100vh - 3.6rem)`；所有尺寸 `overflow-y: auto`。
**效果**: 🔄 待用户验证。
