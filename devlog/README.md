# Exam-in-Mind 开发日志

这个文件夹是项目的开发日志存档。

## 用途

- 记录每个 Phase 的开发过程、关键决策和遇到的问题
- 作为未来整理技术博客和项目复盘的原始素材
- 保留真实的开发轨迹，包括弯路和错误

## 命名规则

每个 Phase 对应一个文件：

```
phase-{编号}-{简短主题}.md
```

示例：
- `phase-1-scaffolding.md`
- `phase-2-brave-search.md`
- `phase-3-models-cache.md`
